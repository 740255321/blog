# 分支操作

# 本地分支操作

WARNING

注意:分支合并首先切换到主干 master,然后合并分支 git merge 分支名

命令 描述
git checkout 分支名 切换到指定分支
git checkout -b 分支名 新建分支并切换到新建分支
git branch -d 分支名 删除指定分支
git branch 查看所有分支,并且*号标记当前所在分支
git merge 分支名 合并分支,注意必须在主干合并分支
git branch -m && -M 旧文件名 新分支名 重命名分支 若分支名已存在,则需要使用-M 强制重命名,否则使用-m 重命名

# 远程分支操作

命令 描述
git branch -a 查看本地与远程分支
git push origin 本地分支名字 推送本地分支到远程
git push origin :远程分支名 删除远程分支(本地分支还在保留)
git checkout -b 本地分支名 origin/远程分支名 拉取远程指定分支并在本地创建分支

# 本地分支冲突

触发条件：在本地分支和主干的同一个文件的相同位置修改了不同内容就会出现冲突 解决方案:根据实际业务需求修改主干冲突文件内容重新提交 git add . get commit -m

# 多人协同操作冲突

触发条件:A 人员与 B 人员同时修改了同一个文件的相同位置并且推送到远 解决方案:修改前先 git pull 拉取远程仓库文件的最新状态再推送

# 打标签

打标签可以理解为版本号

命令 描述
git tag 标签名 新建标签默认为 HEAD
git tag-a 标签名 -m '描述' 添加标签并指定标签描述信息
git tag 查看所有标签
git tag -d 标签名 删除一个本地标签
git push origin 标签名 推送本地标签到远程
git push origin --tags 推送全部未推送过的本地标签到远程
git push origin :refs/tags/标签名 删除一个远程标签
